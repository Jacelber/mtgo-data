# P2-08 Phase 2 Closeout Audit

## Purpose

P2-08 closes Phase 2 by verifying the shared compatibility classifier against the frozen Standard baseline, regenerating every affected public output, and checking the existing MTGO page in a real browser. It does not add archetypes, subtypes, formats, statistical formulas, or front-end features.

No new taxonomy was introduced.

## Deterministic generated-output repair

The initial closeout regeneration found eight byte differences even when the reference date and generated timestamps were held constant. Aggregate counts, rates, matrices, classification results, and statistical formulas were unchanged. The differences came from Python set iteration in three presentation-level tie paths:

- representative-deck distance accumulation;
- equal-delta deck deviation ordering;
- equal-sample matchup archetype ordering.

`stats_standard.py` now sorts card names before distance and difference calculations, uses `math.fsum` for stable distance accumulation, and uses card name as the final deck-difference tie-break. `stats_matchup.py` now uses archetype name as the final equal-sample tie-break. A subprocess regression test runs the relevant calculations under four different `PYTHONHASHSEED` values.

With this repair, 26 generated JSON documents were regenerated under hash seeds `1`, `2`, `17`, and `101` with zero byte differences between runs.

After excluding only the repaired deterministic representative/tie-order fields (`best_deck.deviation_diff`, `average_deck.medoid`, and `archetype_order`), the comparison found zero differences. Metagame totals, high-score and Top 8 counts, match records, win rates, confidence intervals, and matrix values did not change.

A normal regeneration on 2026-07-20 advances the latest complete rolling week from 2026-07-06 to 2026-07-13. That is expected time-window behavior, not a classifier regression. Closeout comparisons therefore used the committed 2026-07-19 reference date and preserved generated timestamps.
