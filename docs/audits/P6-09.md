## Objective

Move Standard onto the same canonical hierarchical matchup calculation as
Modern, prove exact collapsed parent-level compatibility, and expose both
formats through one static front-end renderer with independently expandable
row and column axes.

P6-09 does not add or revise archetype rules, subtype definitions, match-result
handling, confidence-interval formulas, or source-data collection behavior.
P6-10 remains the separately authorized Phase 6 closeout and release task.

## Implementation result

### One shared calculation

`build_all_matchups` now loads hierarchical events for both Standard and
Modern and produces canonical leaf, parent, and parent-to-leaf rollups through
the same code path. Standard's historical `archetype_order`, `overall`, and
`matrix` fields remain temporary compatibility aliases, but they are derived
from the new parent rollup rather than calculated separately.

The Standard and Modern metadata documents now carry the same complete
hierarchy, Pickup-reference, Videre-source, and coverage contract. The
production workflow regenerates the maintained hierarchy catalog for both
formats.

### Shared renderer

The new `assets/js/matchup.js` module treats canonical leaf W-L-D counts as the
source of truth. It builds the requested parent/subtype axis view, sums counts,
and recalculates matches, win rate, Wilson interval, and low-sample state. It
never averages already-calculated percentages.

The MTGO page now:

- defaults both axes to parent archetypes;
- expands rows and columns independently;
- offers one global expand/collapse control;
- determines expandability from maintained taxonomy;
- hides controls for parents with zero or one maintained subtype;
- resets expansion state when the selected format changes;
- exposes Standard and Modern through the same format selector.

Subtype overall records exclude every leaf belonging to the same parent, which
preserves the existing rule that mirror matches are omitted from overall
matchup performance.
