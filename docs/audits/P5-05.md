# P5-05 Normalized Melee Event Assembly Audit

## Scope

P5-05 joins the immutable source records emitted by the P5-04 parser into one
deterministic, Schema-shaped Melee event document. It establishes event-scoped
participant, round, and match identities and verifies cross-resource
references. It does not fetch Melee, interpret round or result semantics,
classify decks, generate statistics, publish JSON, change workflows, or change
the front end.

The approved reference event remains `434455`, and its whitelist entry remains
disabled. The checked-in source snapshot remains synthetic and deidentified.

## Assembly contract

`src/mtgmeta/melee/assembler.py`:

- requires the parsed snapshot event ID and URL to match the reviewed whitelist
  definition;
- creates deterministic event-scoped participant, round, and match IDs from
  source IDs, without using display names;
- links standings, decklist references, decklists, rounds, matches, and match
  competitors through those stable identities;
- rejects missing standings populations, dangling participant or round
  references, conflicting duplicates, multiple ambiguous standings or
  decklists, decklists without references, and decklist-owner conflicts;
- preserves source standings records, original result and status evidence,
  source match and round IDs, raw artifact hashes, and source URLs;
- sorts cross-resource output deterministically and requires an injected,
  timezone-aware `normalized_at` value;
- maps only source card sections needed by the existing normalized-event
  contract and does not classify cards or archetypes;
- leaves participant status, round phase, stage, game format, result type,
  played status, and statistical eligibility unresolved;
- emits explicit `unknown_round_phase` and `unknown_result` issues;
- always returns `quality.status: blocked` and `publishable: false` pending
  P5-06 semantic normalization and later P5-07 publication gates;
- imports no network client and writes no archive or generated output.

The internal phase ID `unresolved` is reserved by the assembler so a whitelist
phase cannot silently collide with the P5-05 placeholder.

## Stable identity behavior

Stable IDs are derived from the source type, Melee event ID, and source record
ID. Consequently:

- the same source participant joins to the same normalized participant across
  standings, decklists, and matches;
- identical display names with different source IDs remain distinct;
- changing a display name does not change the participant ID;
- source page order does not change IDs or the assembled document;
- IDs from different events cannot be joined accidentally.

Identity is event-scoped. P5-05 does not attempt to track a real person across
different tournaments.

## P5-06 boundary

P5-05 deliberately does not interpret strings such as `2-1-0`, `Bye`,
`Dropped`, or a source format label. P5-06 remains responsible for assigning
reviewed stages, round phases, game formats, result types, match points,
played state, and statistical eligibility, including any reviewed
event-specific overrides.
