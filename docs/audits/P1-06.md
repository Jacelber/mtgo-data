# P1-06 Standard Classification Baseline

## Provenance and scope

The original authorized base was `e3f421577f7a11a82d5fd4ddf8709d3fb774adda`; GOV-02 merged at `072b61958a694fae48413fdf747704f8dfec825d`; the pinned active continuation base is `7ac64dbf2460a61685c1894a668a92b782762e35`. GOV-02 changed governance and test-only content; the pinned upstream update added 64 Standard records, taking the corpus from 3872 to 3936. The original subset remained represented by stable `(event_id, loginid)` identities; the additional identities come from the pinned upstream Standard event update.

The earlier corpus and representative tests read `data/standard` at runtime. That was rejected because changing production data could silently change the baseline. All P1-06 classification tests now load only `tests/fixtures/standard/frozen_legacy_corpus.json`, resolve paths from their repository files rather than the process CWD, and reclassify each frozen deck using the unchanged legacy classifier. The stale preliminary `corpus_baseline_*.json` shards were removed.

## Protected behavior and limitations

The baseline freezes observed legacy results without changing classifier code, rules, data, statistics, generated output, public paths, workflows, or dependencies. Protected production hashes were checked at the pinned-base boundary. Future production-data changes cannot expand this corpus automatically; intentional baseline updates require explicit fixture regeneration and review. The fixture retains no player names, uses repository-relative paths, and has no network dependency.
