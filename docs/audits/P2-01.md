# P2-01 Standard Rule Migration Contract Audit

## Purpose

P2-01 defines a frozen, reviewable migration contract for converting the legacy ordered Standard rules into stable archetype, optional subtype, rule, and explicit priority identities.

This task does not change the production rule YAML or classifier. It proves that the proposed explicit identities reproduce the Phase 1 parent-archetype baseline before shared-classifier implementation begins.

## Contract structure

`tests/fixtures/standard/rule_migration_contract.json` freezes:

- 74 stable parent archetype IDs and legacy display names;
- 76 stable rule IDs;
- 76 explicit rule priorities;
- the parent archetype priority derived from its highest-priority compatibility rule;
- four owner-approved optional subtypes;
- a canonical SHA-256 digest of every legacy signature-card condition list;
- cross-platform normalized-content hashes for the production rules, classifier, and frozen corpus;
- observed subtype selection counts and same-parent multi-subtype matches.

Priorities are descending multiples of ten from `760` to `10`. They encode the approved legacy first-match precedence explicitly. Reordering YAML entries will not alter the proposed compatibility choice after the production migration uses these priorities.

## Initial subtype mapping

Only the two existing duplicate legacy archetype groups receive subtypes.

| Parent archetype | Subtype ID | Subtype display name | Selected baseline decks |
| --- | --- | --- | ---: |
| `4-color-control` | `inevitable-defeat` | Inevitable Defeat | 1 |
| `4-color-control` | `rakshasas-bargain` | Rakshasa's Bargain | 4 |
| `izzet-aggro` | `razorkin-needlehead` | Razorkin Needlehead | 8 |
| `izzet-aggro` | `hired-claw` | Hired Claw | 32 |

All other existing Standard archetypes have an empty subtype definition and every associated compatibility rule returns `subtype: null`.

Twenty-one frozen records match both subtype rules beneath the same parent archetype. The higher explicit rule priority selects the subtype while the parent archetype remains unchanged. The future classifier must retain all matched rules for diagnostics rather than hiding this overlap.

## Full-corpus compatibility

Every one of the 3,936 frozen records was evaluated against all 76 legacy rule conditions. The winning proposed rule was selected by explicit priority, with the candidate list deliberately traversed in reverse order to demonstrate that selection does not depend on evaluation order.

Results:

```text
records=3936
rules=76
parent_archetypes=74
unknown=71
multiple_matches=947
maximum_matches_per_deck=3
parent_archetype_differences=0
selected_subtype_decks=45
same_parent_multiple_subtype_matches=21
```

The parent archetype result is identical to the legacy classifier for every record.

- front-end code or behavior.

## Conclusion

The project owner accepted P2-01 and authorized its publication on 2026-07-19. The migration contract is suitable as the reviewed input to P2-02 and later rule migration work. P2-02 remains unauthorized until separately approved after P2-01 is published. New archetypes and additional subtypes remain prohibited during the compatibility-classifier build.
