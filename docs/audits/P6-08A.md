## Historical distinction

The original Standard Videre collector already treated explicit
`400 No results found` and successful empty data as non-failing missing
archives. It did not implement HTTP retries. Its event loop recorded other
exceptions as failures but the original script did not propagate a non-zero
process exit, so a workflow could continue with an incomplete archive.

The generalized MTGO command deliberately fails when any independent event
remains failed. P6-08A preserves this quality gate.

## Authorized behavior

Each individual Videre page request:

- permits three total attempts;
- uses the existing 30-second request timeout;
- waits two seconds between failed attempts;
- retries HTTP 408, 425, 429, every 5xx response, `URLError`, `TimeoutError`,
  and `ConnectionError`;
- immediately preserves `400 No results found` as `NoResults`;
- immediately raises every other HTTP error;
- raises the final original transient exception after attempts are exhausted.

The retry is page-scoped, so a transient failure during pagination does not
require the caller to rebuild or silently accept a partial event. Retry warnings
include the attempt count, original exception, and bounded delay.
