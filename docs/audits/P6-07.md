## Implementation result

Modern receives `weekly_pickup`, `metadata_generation`, and
`catalog_generation` capabilities without changing its `public: false` state.
The hierarchy catalog is generated by the same shared function used by P6-06
matchup documents. It contains:

- 55 parent archetypes;
- 92 most-specific leaves;
- 17 expandable parents;
- zero exactly-one-subtype parents.

Expandability is taxonomy-based: only a parent with at least two maintained
subtypes is expandable. The catalog is independent of observed time windows.

Modern metadata identifies Videre as the matchup source and records:

- 183 admitted official Modern events;
- 161 official events with stored archives;
- 22 official events without stored archives;
- 165 stored archives total;
- four stored archives outside the admitted official statistics set.

The metadata has a null Pickup catalog reference because no week has yet been
manually approved and published.

## Weekly Pickup review boundary

The explicit one-time bootstrap records the 54 stable parent IDs observed in
the latest 12 complete weeks. It refuses to overwrite existing state.

The real latest complete week is 2026-W29 (2026-07-13 through 2026-07-19). Its
review files contain:

- 92 deduplicated Top 8 candidate decklists;
- 92 existing-parent candidates;
- zero new-parent candidates;
- 35 candidates with a selected subtype;
- zero approved candidates.

Existing/new status is keyed by stable parent archetype ID. Subtype ID and name
are informational fields only: they do not split base packs, known-state
membership, or the Pickup population.

Candidate generation does not publish a weekly JSON document or index and does
not update known state. Publication still requires a human to retain selected
rows, set `approved: true`, and supply the review comment.
