## Complete public-source adapter

The initial project-branded User-Agent received `HTTP 403` from `awselb/2.0`.
After owner review raised the acceptance standard, the implementation was
compared with the public `j6e/mtg-meta-analyzer` source at commit
`439a7399c734266fb2e91c9f660fec94a1761477`. The resulting adapter uses ordinary
anonymous browser-compatible `Accept` and User-Agent headers plus the standard
DataTables `X-Requested-With` header. It does not use credentials, cookies,
browser-session material, referer spoofing, CAPTCHA handling, or concurrent
requests.

The complete collector keeps the configured tournament page as its trust root,
deduplicates completed round selectors, selects the final completed numeric
Swiss round for primary standings, paginates public standings and match
DataTables endpoints, and retrieves only decklist GUIDs discovered in those
standings. It rejects redirects, unexpected hosts or paths, invalid identities,
changing page totals, excessive rounds, decklists, responses, or bytes. Raw
manifest contract `2.0.0` records HTTP method, request-body SHA-256, and relevant
round, participant, and decklist source context while retaining parser support
for stored `1.0.0` fixtures.

## Restricted live end-to-end result

The committed whitelist entry remained disabled. For each live validation it
was enabled only in memory and a private task-local raw root was removed after
the anonymous summary was calculated. The successful complete run produced:

- 483 responses: 1 tournament page, 15 standings pages, 105 match pages, and
  362 decklist responses;
- 362 primary standings participants and 362 decklists;
- 19 completed rounds and 2,296 source match records;
- 1,394 verified played Modern Swiss matches eligible for Constructed and
  matchup statistics;
- zero unresolved match results.

The first complete normalization exposed the actual source status vocabulary:
`Active` (1), `Cut` (305), `Disqualified` (1), `Dropped (Self)` (43),
`Dropped (Staff)` (5), and `Eliminated` (7). Evidence-preserving status mapping
now treats normal cut or elimination as completed participation, explicit drops
as dropped participation, and `Disqualified` as its own retained state. A
separate anonymous match
semantics audit found 2,282 two-competitor records without bye or loss reasons,
seven ordinary `Algorithm Assigned` single-competitor byes, and seven
`Qualified` single-competitor records. The latter now normalize as
`awarded_win_top8_lock` only because the reviewed event configuration explicitly
supports that procedure. Of 51 tied matches, four had the explicit structured
`GameWins=(0,0), GameDraws=3` signature and a `0-0-3` source result; these now
normalize to eight per-participant `intentional_draw` results and are excluded
from win-rate and matchup samples. The final complete live run returned the same
source counts, 7 ordinary byes, 7 Top 8 lock awards, and 4 intentional-draw
matches. The one disqualified participant and all 12 affected source matches
remain archived; all 12 are statistically ineligible, including 6 otherwise
eligible Modern Swiss matches. The final sample therefore contains 1,394
eligible Modern Swiss matches. Normalization has zero unresolved issues; final
quality is `warning` with one non-blocking
`disqualified_participant_matches_excluded` issue, and the quality publication
gate passes for the test-only in-memory enabled event.

No real raw archive, normalized participant data, decklist, match record, or
generated output remains in the workspace or repository. The successful quality
gate does not enable the committed event or authorize classification,
statistics, production writes, or publication.
