# P3-03 — MTGO event IO and classification dispatch

## Purpose

P3-03 extracts MTGO page download, event discovery, embedded-JSON parsing, event normalization, safe storage, and classification-rule dispatch from the legacy scripts into format-aware package modules. Standard remains the only executable MTGO format, and the existing Standard compatibility entry points retain their observable data and classification behavior.

- Baseline branch: `master` at `aac20a079b7336d46c2e3bcf2810ac7e56b8710f`.

P3-03 does not fetch live MTGO data during development, change committed event data, enable another format, change a rule or subtype, generate statistics, alter a public JSON contract, modify Videre processing, edit the production workflow, or change the front end.

## New MTGO package boundary

`src/mtgmeta/mtgo/fetch.py` owns injectable page download and retries, exact event-link discovery, embedded event-JSON parsing, completeness checks, deterministic filenames, fetched-record IO, safe event storage, and the explicitly format-selected single-event fetch boundary.

`src/mtgmeta/mtgo/normalize.py` owns the legacy-compatible MTGO event projection, rule loading for an explicitly selected format, and event-deck dispatch to the Phase 2 shared classifier. It does not write classification results into existing event or public-output files.

`src/mtgmeta/mtgo/__init__.py` resolves one format through `configs/formats.yaml`, requires the requested MTGO capability, and resolves all paths beneath the caller-provided repository root. No generalized function defaults to Standard.

## Compatibility boundaries

`batch_mtgo.py` retains its legacy command, month handling, six-link-format recognition, playoff filter, fetched-record behavior, console summary, and output shape. Link recognition is not treated as new-format enablement: the new format-aware fetch function consults the registry and permits only Standard. The legacy multi-format batch behavior remains isolated until the production command and workflow migration in P3-07.

`fetch_mtgo.py` remains a manual compatibility script, including its existing demonstration URL and output projection, but delegates download, parsing, normalization, and storage helpers to the shared package.

`classify_standard.py` retains the Phase 2 string and result APIs. It now resolves the Standard rule and event paths from the registry. The full 3,936-deck frozen corpus continues to select the same parent archetypes and approved subtypes.

The P3-01 hard-coded inventory remains a historical baseline. Its executable test now requires the migrated P3-03 path snippets to be absent while continuing to require every later-phase coupling boundary to remain visible until its assigned task.

## Side-effect safety

The new single-event fetch boundary validates the format, required capabilities, URL format, and repository paths before making a request or creating a directory. Disabled, decision-gated, unknown, and URL-mismatched formats therefore fail before network or filesystem effects. Development tests inject fake requests and temporary roots; no test accesses MTGO.
