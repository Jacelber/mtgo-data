# P1-13 Embedded Public Schema Version Migration

## Purpose

P1-13 completes the producer migration deferred by P1-08. Every Standard MTGO public JSON declared by `schemas/manifest.json` now identifies compatibility contract version `1.0.0` in a top-level `schema_version` field.

## Scope

The manifest resolves to 17 tracked public files:

- four range statistics files;
- four representative-deck files;
- the range catalog;
- four matchup files;
- the matchup catalog;
- metadata;
- the Weekly Pickup catalog;
- one published Weekly Pickup report.

Legacy files outside the manifest, including `matchup_matrix.json`, `matchup_overall.json`, and `pickup/known_archetypes.json`, are unchanged.

## Producer contract

`public_contract.py` is the single source of `PUBLIC_SCHEMA_VERSION`. The Standard statistics, matchup, metadata, and Weekly Pickup public producers use its `versioned()` helper. The helper returns a new mapping, places `schema_version` first, and rejects accidental double versioning.

All eight output schemas require `schema_version` and constrain it to `1.0.0` through the shared common schema. The manifest records that embedded versions are required. Validation reports the count of embedded versions.

## Compatibility verification

The migration was checked against the task base commit by loading each of the 17 files from Git, removing only `schema_version` from the migrated document, and comparing the complete parsed JSON values. All 17 files were equal after removal of that field.

The P1-11 public contract tests continue to validate paths, catalogs, periods, aggregate counts, matchup reciprocity, and Weekly Pickup references. The added field is ignored by the current front end; no fetch path or rendering logic changed.

## Boundaries

P1-13 does not change schema version semantics beyond making the existing `1.0.0` contract explicit. It does not change statistics, classification, public paths, front-end code, workflows, legacy non-contract JSON, or production inputs, and it does not dispatch the production update workflow.
