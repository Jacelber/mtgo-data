# P3-02 — Format registry and safe path resolution

## Purpose

P3-02 creates the validated configuration boundary required before MTGO implementation code can become format-aware. It records every known format, keeps Standard as the sole executable MTGO format, and provides explicit lookup and repository-relative path resolution without selecting a default format.

## Baseline and scope

- Baseline branch: `master` at `2bab92f54cff915f4730d64e4227bb43cc475029`.
- Baseline recovery tag: `phase-2-shared-classifier-baseline`.
- Task branch: `codex/p3-02-format-registry`.
- Allowed final paths: `configs/formats.yaml`, `schemas/formats.schema.json`, `src/mtgmeta/config.py`, `tests/`, this audit, `docs/ROADMAP.md`, and `docs/STATUS.yaml`.

The task does not modify generators, workflow files, public JSON, classification rules, statistics, front-end code, source data, or generated data. `schemas/manifest.json` remains unchanged because it is the existing public-generated-JSON mapping, not a configuration registry.

## Registry contract

`configs/formats.yaml` is manually maintained and versioned. It contains the six P3-01 known IDs:

- `standard`: executable on MTGO and publicly available;
- `pauper`, `modern`, `pioneer`, and `legacy`: planned and disabled;
- `vintage`: decision-gated and disabled.

Every entry declares its display name, lifecycle state, public availability, MTGO enablement, supported MTGO capabilities, and format-specific repository-relative paths for events, matches, rules, statistics, and reports. Planned paths are declarative only; P3-02 does not create their directories or read their files.

`standard` preserves the P3-01 contract paths exactly: `data/standard`, `data/standard/mtgo/matches`, `my_archetypes/standard.yaml`, `stats/standard/mtgo`, and `reports/standard/mtgo`.

## Safety behavior

The registry API requires a caller-provided format ID. It has no default-format parameter. Unknown IDs raise `UnknownFormatError`; known disabled IDs raise `DisabledFormatError`; Vintage's error identifies its decision-gated state.

The resolver accepts only normalized repository-relative forward-slash paths. It rejects Windows drive paths, absolute paths, backslashes, and traversal components before resolving under the provided repository root. Each configured path must also retain its own format ID, preventing a configured format from pointing at another format's storage tree.
