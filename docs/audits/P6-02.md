## Pre-change findings

The shared classifier itself was already format-independent, but the surrounding diagnostic integration had three incomplete boundaries:

1. `build_classification_reports` defaulted its metadata to Standard and accepted an explicit format unrelated to the supplied rule set.
2. The generic JSON event loader did not compare an event's embedded MTGO format with the requested project format. The current `data/modern/` directory contains 181 `CMODERN` files and two misplaced `CPREMODERN` files.
3. The only complete report-generation command correctly required an executable product format, so Modern could not be integration-tested through the real archived-event and diagnostic path without either enabling the product prematurely or bypassing the path in tests.

## Implementation

`src/mtgmeta/mtgo/classification.py` adds a read-only audit boundary:

- it requires an explicit format known to the collection-enabled registry path;
- resolves only that format's configured event and rule paths;
- loads the exact shared rule file and requires its declared format to match;
- requires a non-empty embedded format on every event;
- includes only the expected MTGO marker, `CMODERN` for Modern;
- returns cross-format documents as structured exclusions;
- builds existing de-identified diagnostic objects in memory;
- checks the result for forbidden participant identity fields;
- never creates a report, statistics, or public-output directory.

The production report writer remains product-gated. It now uses the same embedded-format loader but treats any cross-format input as a fatal error before writing. This preserves the stricter production boundary while allowing a planned format's explicit read-only audit to measure known exclusions.

`build_classification_reports` now derives its default format from the supplied rule set and rejects an explicit mismatch. This removes an implicit Standard label and prevents correctly classified data from being published with unrelated format metadata.
