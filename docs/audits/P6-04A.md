## Question

Can the current Modern MTGO event collector also select Premodern events, causing unsupported `CPREMODERN` data to be stored under `data/modern/`?

## Static boundary

`fetch_event_months` calls `discover_event_links` with a one-item recognized-format collection containing only the explicitly requested format. `parse_event_link` then:

1. extracts the URL path;
2. removes the `/decklist/` prefix;
3. reads the complete token before the first hyphen;
4. returns it only when it exactly matches the recognized-format set.

For a Modern run, `/decklist/modern-challenge-*` resolves to `modern`, while `/decklist/premodern-challenge-*` resolves to `other`. Links containing `league` are separately rejected. Output routing uses the already authorized Modern context rather than substring inference.

The focused regression `test_link_discovery_is_exact_and_does_not_confuse_premodern_or_leagues` already freezes these cases. It asserts that a Modern challenge is included, a Modern league is excluded, and a Premodern challenge is classified as `other`.

## Live public-page evidence

The current discovery function was run anonymously against the same public monthly archive pages used by production collection, with ordinary project headers and no credentials, cookies, or stored response data.

The public MTGO pages use distinct `modern-*` and `premodern-*` URL prefixes, so the current exact-token parser keeps the two formats separate.

## Historical finding

The two Premodern documents removed in P6-04 were added by initial database commit `aba1ed4213ef573aff956fbda17623546f082396`. The tracked crawler had already been introduced by `d6cb867ec4079d6cf16ca082e073555336828fcf` with an explicit comment and implementation preventing `premodern` from matching `modern`. The repository therefore does not show those files being produced by the current or tracked historical crawler; their pre-repository import provenance is not recorded.

## Result

No correction is required. The current Modern monthly collection candidate set excludes Premodern links, and the existing regression test protects the exact boundary. No source code, configuration, workflow, production data, or test behavior changed in P6-04A.
