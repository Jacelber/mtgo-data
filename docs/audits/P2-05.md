# P2-05 Full-Match Shared Classifier Audit

## Purpose

P2-05 implements the format-independent classification decision core required after the P2-04 Standard rule migration. It evaluates every rule, selects by explicit priority, attaches the optional subtype of the selected rule, and retains deterministic diagnostics for Unknown, multiple-match, overridden-match, and equal-priority conflict cases.

This task does not route the legacy Standard production entry points through the shared classifier. That integration remains P2-06.

## Shared classification model

`src/mtgmeta/classifier.py` adds immutable models for:

- condition evidence, including zone, observed count, and configured bounds;
- every matched rule with parent archetype, optional subtype, rule ID, and priority;
- the complete classification result;
- top-priority matches;
- overridden lower-priority matches;
- conflict matches and conflict kind;
- sanitized invalid-input errors.

Classification statuses are:

- `classified`;
- `unknown`;
- `conflict`;
- `invalid_deck`.

The result retains parent archetype ID and display name, nullable subtype ID and display name, selected rule ID and priority, all matched rules, and the diagnostics required for later P2-07 reporting.

## Full-match and deterministic selection

Every rule is evaluated before selection. Matching rules are ordered by explicit priority followed by stable identities, so archetype and nested rule collection order do not affect the result.

When one identity has the highest priority, it is selected and all lower-priority matches remain in `overridden_matches`. A deck matching multiple rules is therefore not automatically a conflict.

When no rule matches, the result is `unknown` with nullable selected fields.

## Equal-priority semantics

P2-05 distinguishes semantic conflicts from equivalent rule ties:

- equal-highest matches pointing to different parent archetypes produce a `parent_archetype` conflict and no selected parent;
- equal-highest matches pointing to the same parent but different subtype identities produce a `subtype` conflict, retain the unambiguous parent, and leave subtype and selected rule unresolved;
- equal-highest matches pointing to the same parent and subtype remain `classified`, use stable rule ID ordering for the selected rule, and set the priority-tie diagnostic.

No current Standard compatibility rule shares a priority with another Standard rule. The explicit conflict tests use controlled rule models and do not alter production rules.

## Boundaries and next step

P2-05 adds the shared classifier API but does not replace the P2-04 legacy adapter. It does not generate machine-readable diagnostic report files; P2-07 owns those artifacts. It does not add archetypes, subtypes, formats, statistical behavior, or front-end presentation.

P2-06 must not begin until the project owner accepts P2-05, P2-05 is published, and P2-06 receives separate authorization.
