## Purpose and boundary

P6-04 turns the reviewed Modern taxonomy into a reproducible classification-report product without enabling the rest of the MTGO Modern product. Modern becomes executable and non-public with only `classification`; statistics, matchup data, Weekly Pickup, metadata, catalogs, workflows, public format discovery, and front-end behavior remain disabled.

P6-03 publication metadata was closed first: implementation commit `b313ea5ccbb0b3dee59fdaaaf00193889de228bb`, pull request #81, and merge commit `ccdd2495ed1bbf8029c0d588337220574a3a9b63` passed remote repository validation on 2026-07-22.

## Implemented contract

- Format registry Schema remains `1.1.0`; Modern is enabled through its existing capability contract.
- Modern is executable, non-public, and declares only the classification capability.
- The two misplaced Premodern documents detected in P6-01/P6-02 are deleted from `data/modern/`; Premodern is not a supported product format.
- Any future cross-format document blocks generation before report writes; there is no exclusion allowlist.
- Classification-report Schema remains `1.0.0`; its format and source-file constraints now cover format-scoped diagnostics.
- The Schema manifest now covers 29 generated JSON documents: the existing 23 Standard documents and six Modern classification documents.
