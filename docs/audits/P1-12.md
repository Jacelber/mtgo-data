# P1-12 Standard Classification Quality and Repository Hygiene Baseline

## Purpose

P1-12 makes legacy Standard classification quality measurable without changing classification results. It also prevents common runtime, dependency, operating-system, log, backup, and temporary artifacts from becoming tracked repository content.

## Frozen quality baseline

The self-contained P1-06 corpus and the unchanged 76-rule legacy classifier produce:

- 3,936 records;
- 71 Unknown results;
- 947 records that match more than one rule;
- at most three rule matches for one record;
- two duplicated display names: `4-Color Control` and `Izzet Aggro`.

`tests/fixtures/standard/quality_baseline.json` stores aggregate values and a deterministic digest of anonymous corpus identities plus matching rule names. It contains no player names or logins.

The legacy classifier resolves multiple matches by the first matching YAML entry. The rules do not yet have stable rule IDs or explicit priorities. Therefore P1-12 calls these results `multiple_matches`; it does not mislabel them as equal-priority conflicts under the future classifier contract.

## Existing production diagnostic risks

The scheduled workflow currently regenerates and commits `unknown_highperf.txt` and `unknown_clusters.txt`. The legacy generators include player display names, and their explanatory text is mojibake in the current files. These reports are not public JSON consumed by `index.html`, but they are tracked production diagnostics.

P1-12 records this issue without rewriting the generators or generated reports. A later authorized migration should define an anonymized, versioned generated-report format and update its workflow publication contract together. Manually editing the current reports would leave the generators incorrect.

## Repository hygiene

`validate_repository.py` now rejects tracked paths containing runtime caches, test caches, task virtual environments, dependency directories, operating-system artifacts, logs, temporary files, editor swaps, or backups. `.gitignore` covers the corresponding common local artifacts.

Generated diagnostic reports are not treated as temporary files merely because they are text files. Their publication and privacy contract must be handled explicitly rather than hidden by an ignore rule.

## Commands

```text
python validate_standard_quality.py
python validate_repository.py
python -m pytest tests/test_standard_quality_baseline.py tests/test_validate_repository.py
```

## Boundaries

P1-12 does not change classification rules, YAML order, legacy first-match resolution, archetype assignments, statistics, public JSON, front-end behavior, report generators, generated reports, or GitHub Actions. It does not implement rule IDs, priorities, the shared classifier, another format, or Tabletop Major Events.
