# P3-05 Videre Matchup and Classification-Report Routing Audit

## Purpose

P3-05 moves the remaining Standard-only Videre match fetching, matchup generation, and MTGO classification-report routing behind the validated Phase 3 format registry. Standard remains the only executable format. The task changes no statistical formula, public JSON contract, schema, rule, workflow, Weekly Pickup, metadata, catalog, or front-end source.

## Implementation

`src/mtgmeta/mtgo/matchup.py` now owns format-aware Videre pagination, fetched-event selection, safe match storage, official-event classification mapping, physical-match deduplication, mirror and non-mirror accumulation, rolling-window matchup generation, and matchup catalog generation.

The shared interfaces require an explicit repository root and format ID. They authorize `matchup_statistics` through `configs/formats.yaml` before network access, directory creation, event reads, match reads, rule reads, or output writes. Unknown formats, disabled formats, missing capabilities, and unsafe non-numeric Videre event IDs fail before network or output side effects.

The existing root commands remain Standard compatibility entry points:

```text
fetch_videre_matches.py
stats_matchup.py
generate_classification_reports.py
```

`generate_classification_reports.py` now resolves default event, rule, and report paths through the registry. `src/mtgmeta/reports.py` receives explicit format and source metadata while retaining Standard defaults for existing callers. Report identity removal, stable pseudonymous deck IDs, Unknown evidence, conflict handling, strict validation, and subtype diagnostics are unchanged.

The production workflow is intentionally unchanged; command orchestration migration belongs to P3-07.

## Statistical and privacy preservation

The matchup implementation preserves the existing rules:

- one physical match is counted once using event, round, and unordered player identity;
- byes and malformed results are excluded;
- a match is dropped when either deck cannot be mapped;
- mirror cells retain the existing two-perspective record;
- non-mirror cells and overall records remain symmetric;
- overall records continue to exclude mirror matches;
- win rate, Wilson half-width, low-sample threshold, range boundaries, ordering, and JSON fields are unchanged;
- primary matchup statistics continue to aggregate the parent archetype rather than subtype.

The six classification reports remain de-identified. No player name, login ID, or raw player record is written to report output.
