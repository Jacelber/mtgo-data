## Automated closeout contract

The Phase 6 closeout tests intentionally aggregate stable contracts rather than
freeze mutable live counts. They verify:

- Standard and Modern are the only complete public products;
- the production product and hierarchy loops match that registry state;
- both formats expose statistics, matchup, hierarchy, metadata, and diagnostic
  layers for 1-, 4-, 12-, and 36-week windows;
- every canonical leaf W-L-D count rolls up exactly to the emitted parent
  matrix and to twice the physical counted-match total;
- parent overall records exclude parent mirrors;
- Standard compatibility aliases are exact projections of the stable-ID parent
  representation;
- Modern does not emit the temporary Standard compatibility aliases;
- the public format selector uses the shared hierarchical renderer.

## Front-end acceptance repair

Browser closeout exposed one P6-09 interaction gap: expanding an individual
row or column replaced its parent node with subtype nodes and therefore removed
the only per-axis collapse button. The statistical view was correct and the
global collapse control remained available, but individual collapse was not.

The focused repair marks only the first rendered subtype in each expanded
parent group as the location for that parent's axis toggle. The button keeps
the parent label and collapses only its row or column axis. Other subtype rows
do not duplicate the control, and parents with fewer than two maintained
subtypes remain non-expandable. No generated data, formula, taxonomy, or
public path changes.
