# P3-04 Format-Aware MTGO Statistics Audit

## Purpose

P3-04 migrates Standard MTGO event and rolling-range statistics into a format-aware package boundary without changing statistical semantics or published Standard output. It does not enable another format or migrate Videre matchup processing, Weekly Pickup, metadata, catalogs, the production workflow, or the front end.

## Implementation

`src/mtgmeta/mtgo/stats.py` now owns the existing MTGO event and rolling-range calculations. The caller must provide an explicit repository root and format ID. Registry capability checks and safe path resolution occur before event reads or output-directory creation. Standard is the only executable format; a disabled format fails before output side effects.

The migrated capabilities include theoretical-round lookup, high-score thresholds, parent-archetype aggregation, latest-complete-week selection, 1/4/12/36-week windows, average points per theoretical round, Top 8 and high-score fields, representative decks, four-week average-deck bases, Core/Flex cards, construction deviation, and recent construction change.

`stats_standard.py` remains the production command and compatibility wrapper. It preserves the legacy Standard classifier call path and the helper imports consumed by `stats_matchup.py`, `weekly_pickup.py`, `cluster_unknown.py`, `dump_unknown_highperf.py`, and regression tests. The production workflow still runs `python -B stats_standard.py`.

The construction-statistics path deliberately retains the two legacy public-output card aliases instead of adopting the broader shared OM1/SPM mapping. That compatibility boundary prevents public deck names and deviation values from changing during this refactor. Statistics remain grouped by parent archetype; subtype identity does not change public aggregation.
