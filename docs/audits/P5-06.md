# P5-06 — Melee semantic normalization audit

Date: 2026-07-21
Branch: `codex/p5-06-melee-semantic-normalization`

## Scope

P5-06 converts the deterministic P5-05 assembly into reviewed semantic records.
It resolves round phase, stage, actual format, participant status, result type,
match points, and primary Constructed and matchup eligibility. It does not fetch
Melee, classify Modern decks, generate statistics, publish data, change the
front end, or modify production workflows.

## Evidence contract

- Per-competitor outcome and point fields are retained when present.
- Competitor order is never treated as winner evidence.
- Played matches require two identified competitors and a consistent win/loss
  or draw/draw result with compatible points.
- The legacy identity-only stored shape remains parseable but cannot become a
  played result without additional evidence.
- Whitelist Schema 3.0.0 provides explicitly verified, event-scoped match
  overrides with a reason and HTTPS evidence.
- Overrides fail on missing matches, invented participants, duplicate targets,
  inconsistent played state, or unsupported Top 8 lock awards.

## Eligibility contract

Only real played matches in a resolved Constructed Swiss phase whose actual
format equals the event's configured Constructed format are eligible for both
primary Constructed statistics and matchup matrices. Draft, playoff, bye,
intentional draw, no-show, unplayed drop, administrative, awarded, and unknown
results are retained but ineligible.

Unknown phase, participant status, or match result creates a blocking quality
issue. A fully resolved P5-06 document may have quality status `valid`, but it
always remains `publishable: false`; P5-07 owns final publication blocking.

## Compatibility and boundaries

The normalized event Schema remains 1.0.0. Stable event-scoped identities and
P5-05 assembly behavior are preserved. The stored fixture is synthetic and
deidentified. No network call or production-data write occurred.
