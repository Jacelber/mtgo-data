# P3-07 Generalized MTGO Command and Workflow Migration Audit

## Purpose

P3-07 completes the production-entry migration for the format-aware MTGO internals built in P3-02 through P3-06. It adds one explicit command surface and moves the sole scheduled production workflow away from Standard-only root commands without enabling another format or changing statistical semantics.

## Command contract

`src/mtgmeta/mtgo/__main__.py` requires an explicit `--format` and exposes commands for official event fetching, Videre match fetching, rolling statistics, matchup statistics, Weekly Pickup candidate generation and manual publication, metadata, and de-identified classification reports. Unknown formats and formats without collection permission fail before event-fetch network or output side effects. Every non-collection product command still requires an executable format and its declared capability.

Standard remains the only executable product format. DEC-033 separately preserves official event archival for Standard, Pauper, Modern, Pioneer, Legacy, and Vintage. Non-Standard Videre collection is not added. The root-level Standard commands remain available for import and command compatibility, but the production workflow no longer depends on them.

## Event-fetch migration

The format-aware fetch module now owns the current-and-previous-month production loop. It authorizes `event_collection_enabled` before reading the fetched ledger or accessing the network, discovers links only for the requested format, preserves the playoff-only event filter, writes through registry-resolved event paths, and records failures in a command summary with a nonzero exit status.

This closes the legacy `batch_mtgo.py` boundary without dropping its six-format raw archive. A collection-enabled planned format remains prohibited from classification, statistics, Videre, Pickup, or publication. No live MTGO request was made during P3-07 development.

## Workflow migration

`.github/workflows/update.yml` remains the only scheduled production workflow. Its daily `20:00 UTC` schedule, manual dispatch trigger, master-only guard, write permission, concurrency group, timeout, pinned dependency installation, validation sequence, generated-data commit, and direct master publication remain intact.

The workflow sets `PYTHONPATH=src`, declares the six registry-approved `MTGO_EVENT_FORMATS`, and runs the generalized event command once per archive format. It then uses `MTGO_FORMAT=standard` for Videre and every product-generation step. Pickup candidate preservation is handled by the format-aware command instead of a Standard-path shell fragment. Manual Pickup publication is not scheduled.

The legacy `dump_unknown_highperf.py` and `cluster_unknown.py` steps are no longer scheduled. Their identity-bearing text outputs were superseded by the deterministic, de-identified, Schema-versioned P2-07 classification reports. The legacy scripts and historical files are not deleted in this task.

## Compatibility boundaries

P3-07 changes production orchestration and version 1.1.0 of the internal format-registry Schema, not public product-data contracts. It changes no archetype or subtype rule, statistical formula, public JSON schema, front-end source, committed event data, committed statistics, or committed report document. Videre remains Standard-only and remains in the update sequence before matchup generation. Weekly Pickup retains its human approval boundary.

P3-08 remains responsible for the Phase 3 fixed-reference and front-end closeout. Legacy root commands are therefore retained until that separate review is complete.

## Validation

The focused format-registry, command, event-I/O, statistics, matchup, Pickup, migration-contract, production-workflow, and Schema suite passed 88 tests. The full suite passed 223 tests. Repository, rule, Standard quality, classification-report, and JSON Schema validation passed. Fixed-reference statistics, matchup, Pickup, metadata, and classification-report regression tests remained unchanged and passed. Collection-disabled formats were verified to fail before network, ledger, or output side effects; collection-enabled planned formats were verified to write only their own raw-event path and remain blocked from product execution.

No production workflow was dispatched, no live MTGO or Videre fetch was performed, and no generated production file changed.

## Stop point

P3-07 local implementation is complete. Commit, push, pull-request creation, merge, workflow dispatch, production fetches, and P3-08 require separate project-owner authorization.
