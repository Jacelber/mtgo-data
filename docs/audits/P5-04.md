# P5-04 Stored Melee Source Parser Audit

## Scope

P5-04 converts already stored P5-03 raw-response snapshots into immutable source-level records. It does not fetch a source, assemble cross-resource participant identities, create a normalized event, assign event stages or game formats, normalize result types, classify decks, generate statistics, publish public JSON, or change the front end.

The approved reference event remains `434455`, and its whitelist entry remains disabled. No live Melee request was made and no real raw archive was created during this task.

## Stored fixture

`tests/fixtures/melee/source_snapshot/` is a synthetic and deidentified P5-03-compatible snapshot. It contains:

- one tournament HTML response with one supported embedded JSON payload;
- two standings;
- one decklist reference;
- three source rounds, including a source label with no numeric round;
- two source matches, including a one-participant source record;
- one JSON decklist with two source card rows;
- a raw-archive manifest with source paths, sizes, and SHA-256 values.

Fixture identities and card names are synthetic. The fixture is not represented as a live capture of the current Melee site.

## Parser contract

`src/mtgmeta/melee/parser.py`:

- verifies that `manifest.json` uses the P5-03 raw archive contract;
- rejects missing, symlinked, nested, absolute, traversing, or duplicate response paths;
- verifies each stored response byte count and SHA-256 before parsing;
- accepts UTF-8 JSON or exactly one supported `application/json` payload in stored HTML;
- rejects duplicate JSON keys, invalid UTF-8, malformed JSON, oversized responses, resource/content mismatches, missing required fields, unsupported fields, malformed primitive values, duplicate record IDs, and event-ID mismatch;
- emits immutable source-level records for tournaments, standings, decklist references, rounds, matches, decklists, and cards;
- preserves source IDs, labels, result text, status text, and format text without converting them into normalized semantics;
- imports no network client and performs no archive write.

The parser intentionally does not decide whether a round is Day 1, Day 2, Draft, Constructed, or playoff. It does not convert source match text into played win, loss, draw, intentional draw, bye, no-show, awarded win, or another normalized result. Those responsibilities remain in P5-05 and P5-06.

## Known limitation

P5-04 proves deterministic parsing of the reviewed stored fixture contract. It does not prove that the current live Melee site exposes identical HTML or JSON shapes. Real-source compatibility remains deferred to the separately authorized P5-08 live validation path. A future reviewed source shape must update the stored fixture and parser tests rather than being silently ignored.
