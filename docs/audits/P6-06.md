## Implementation

- Added `matchup_statistics` to Modern's executable capabilities.
- Added exact-format Modern event loading with visible cross-format rejection.
- Retained stable parent IDs and selected subtype IDs for both competitors.
- Represented subtype leaves as stable composite IDs such as
  `broodscale-combo/gruul-broodscale`; a parent with no subtype definitions is
  its own non-expandable leaf.
- Blocked a classified deck when its parent defines subtypes but none is
  selected. No residual, `Other`, or `Unspecified` subtype is synthesized.
- Added canonical directed leaf-level W-L-D accumulation. Every counted
  physical match creates exactly two inverse observations.
- Added independent row-axis and column-axis count aggregation so
  subtype-to-parent and parent-to-subtype views are directly calculable from
  the same canonical counts.
- Generated the parent matrix by summing leaf counts. Rates are calculated
  only after aggregation; percentages are never averaged.
- Corrected draw handling to use `wins + 0.5 × draws` for both win rate and the
  Wilson interval input. The committed Standard corpus contains no matchup
  draws, so its public files remain byte-identical.
- Added separate Standard legacy-name and Modern stable-ID branches to the
  matchup Schemas.
- Added a disposable `--output-dir` option for classification-report testing so
  integration tests no longer rewrite committed production reports.

Videre supplies both competitors' directed rows for the same match. Those
duplicates are collapsed to one physical match before two canonical inverse
observations are emitted. A match is counted only when both competitors have an
official published decklist and both decks classify. This preserves the
existing MTGO matchup population boundary.
