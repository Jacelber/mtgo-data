# P1-11 Standard Public Output and Front-end Compatibility Baseline

## Purpose

P1-11 closes the gap between per-file JSON Schema validation and the current Standard MTGO page. It records the page's public path contract and adds read-only checks for relationships that JSON Schema cannot express across files.

## Baseline

- Product: MTGO Environment Trends
- Format and source: `standard` / `mtgo`
- Front end: `/index.html`
- Range set: 1, 4, 12, and 36 weeks
- Public roots: `/stats/standard/mtgo/` and `/stats/standard/mtgo/pickup/`

The frozen fixture contains stable path templates and range identities, not generated measurements. Normal scheduled updates may change dates, archetypes, decklists, and counts while retaining this contract.

## Automated guarantees

`tests/test_standard_public_contract.py` verifies:

- every path template used by the page remains present;
- every required Standard range, deck, matchup, metadata, and pickup index file exists;
- statistics and matchup catalogs reference existing files;
- catalog metadata agrees with target documents;
- range and deck periods agree;
- range archetype counts reconcile to total decks;
- matchup cells reconcile internally and reciprocally;
- matchup catalog counts reconcile to matrix counts;
- pickup index metadata and item counts agree with each weekly report.

These tests complement, rather than replace, the P1-08 JSON Schemas.

## Manual guarantee

`docs/checklists/STANDARD_FRONTEND_SMOKE.md` defines the repeatable browser smoke check for rendering and interaction behavior that static tests cannot prove.

## Boundaries

P1-11 does not change `index.html`, generators, statistical formulas, classification rules, workflows, schemas, or generated public JSON. It does not authorize additional formats or the Tabletop Major Events product.
