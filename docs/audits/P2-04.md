# P2-04 Standard Rule Migration Audit

## Purpose

P2-04 migrates the production Standard classification rules from the legacy ordered YAML shape to the versioned P2-03 rule model. The migration adds stable archetype IDs, globally stable rule IDs, explicit priorities, and only the four owner-approved compatibility subtypes while preserving every Phase 1 parent-archetype result.

## Migrated production contract

`my_archetypes/standard.yaml` now uses classification-rule schema version `1.0.0` and format ID `standard`.

The migrated file contains:

- 74 parent archetype definitions;
- 76 classification rules;
- 74 stable archetype IDs;
- 76 globally unique stable rule IDs;
- explicit rule priorities from `760` through `10` in descending multiples of ten;
- exactly four subtype definitions under the two approved duplicate legacy parent groups.

Every identity, display name, priority, subtype reference, and legacy condition digest is checked against `tests/fixtures/standard/rule_migration_contract.json`.

## Legacy compatibility boundary

`src/mtgmeta/legacy_rules.py` provides a temporary adapter from the immutable shared `RuleSet` to the flat rule objects consumed by existing Standard entry points. The adapter sorts rules by explicit priority and returns only the legacy parent display name and signature-card shape.

The production `classify_standard.load_rules()` entry point now loads and validates the versioned configuration, then applies this adapter. Matching and first-match classification functions remain unchanged. Subtype selection and full-match diagnostics remain P2-05 work; shared-classifier integration remains P2-06 work.

Tests deliberately reverse both archetype and nested rule collections and confirm that the adapter still produces the same compatibility order. YAML collection position therefore no longer defines Standard rule precedence.
