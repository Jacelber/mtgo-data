# P6-01 Modern Rule Compatibility Baseline Audit

## Purpose

P6-01 establishes the first Modern parent-archetype baseline for the shared classifier. It adapts a pinned public j6e taxonomy, encodes the source winner semantics as stable identities and explicit priorities, and freezes a de-identified full-corpus comparison before any Modern taxonomy extension or product activation.

## Reference semantics

The upstream deterministic signature classifier:

1. counts normalized front-face names from the mainboard only;
2. requires every signature-card condition to match;
3. selects the matched definition with the greatest signature-condition count;
4. preserves the earlier source-list definition when those counts tie.

The shared classifier selects the unique highest rule priority. P6-01 therefore assigns each migrated rule:

```text
priority = condition_count * 1000 + (38 - source_index) * 10
```

This exactly represents the two-level upstream order while remaining independent of YAML collection order. Every migrated card condition explicitly uses `zone: main`.

The upstream centroid pass is intentionally outside this migration contract. Its labels depend on the surrounding corpus, whereas the project requires unmatched decks to remain explicit, measurable, and reviewable `Unknown` results. This boundary was owner-approved before implementation and is recorded in DEC-039.

## Change boundaries

P6-01 changes only:

- `my_archetypes/modern.yaml`;
- the Modern frozen corpus and migration contract;
- Modern migration tests;
- DEC-039, this audit, ROADMAP, and STATUS documentation.
- the repository notice required for direct CC BY 4.0 attribution.

P6-01 does not:

- change `configs/formats.yaml` or enable Modern;
- classify production Modern data;
- generate Modern statistics, reports, matchups, metadata, or public JSON;
- change Standard rules, data, statistics, or public behavior;
- change Melee data or behavior;
- change workflows or either front end;
- add new archetypes or subtypes.

P6-02 remains separately controlled. It completes the remaining shared-classification and diagnostic integration while preserving all 38 P6-01 parent rules and defining no subtype. P6-03 separately owns the first review of Unknown decks, overlaps, rule extensions, new parent archetypes, and optional subtypes.
